# Node and Lock Model

## Node classes

- Mission Node.
- Idea Node.
- Functional Spec Node.
- Research Need Node.
- Evidence Resource.
- Decision Node.
- Architecture Node.
- Code Object Node.
- Build Unit.
- QA Gate.
- Handoff Packet.
- Reusable Pattern.

## Core Lock

Protects mission, target user, success criteria, non-goals, protected surfaces, and final product direction.

## Contract Lock

Protects shared interfaces, data contracts, metric definitions, handoff contracts, and cross-team boundaries.

## Domain Lock

Protects team-owned decisions inside product, design, engineering, data, or QA lanes.

## Evidence Lock

Protects evidence used for decisions and claims.

## Phase Lock

Protects release sequencing and phase boundary.

## Change rules

Locked content cannot be silently changed.

Every proposed change must show affected node, proposed change, rationale, evidence, owner, risk, review gate, and fallback.
