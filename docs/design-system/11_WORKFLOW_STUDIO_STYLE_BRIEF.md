# 11 — Workflow Studio Style Brief

## Direction

OpenClaw should feel like an operator workflow studio: artifact-first, evidence-first, governed, and compact. It may be Dify-inspired in layout patterns but must not copy Dify branding, assets, logos, exact surfaces, or proprietary visual identity.

## Visual principles

| principle | rule |
|---|---|
| canvas-first | Workbench/object-map screens should put the workflow/object canvas at visual center. |
| operator nodes | Node cards should show object type, state, inputs, outputs, evidence, and guardrails. |
| palette left | Safe object/node palette belongs on the left or as a collapsible rail. |
| inspector right | Selected object details belong in a right inspector/drawer. |
| quiet governance | Governance is visible but not visually louder than work artifacts. |
| panel density | Use compact panels, subtle borders, and information-dense rows. |
| status chip hierarchy | Status chips should be readable, semantic, and secondary to artifact labels. |
| trace/evidence foreground | Evidence and trace links should be near decisions, findings, and handoff readiness. |

## Layout guidance

- Use light neutral workflow canvas.
- Use subtle dotted or gridded background only if it improves orientation.
- Keep cards compact and scannable.
- Use clear selected/focus states for nodes.
- Use consistent left palette, center canvas, right inspector composition.
- Keep safety copy trigger-adjacent, not bloated.

## Anti-patterns

- Dark command center unless Point approves.
- Generic SaaS hero layout.
- Fake live execution controls.
- Excessive decorative gradients.
- Copying Dify branding, logos, assets, exact colors, or exact product surfaces.
- Hiding mock/static honesty.
- Status colors without labels.
