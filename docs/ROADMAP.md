# Roadmap

This document provides a high‑level roadmap for the OpenClaw Cooperative Cockpit project. Major milestones include:

1. **Repository setup** – Establish agent workflows, approval and concurrency policies, and validation scripts. *(Complete)*
2. **Static MVP assimilation** – Bring the static HTML/CSS/JS prototype into the repo under `apps/static-mvp/` and record the handoff. *(Complete)*
3. **Static MVP product lock and QA closeout** – GOAL-004 through GOAL-008 normalized the object model, UI mapping, Work Packet/Handoff Packet preview, and QA evidence. *(Complete)*
4. **Canon reconciliation** – Align current-state docs, operating authority, object terminology, readiness semantics, and package manifests so agents can trust the repo. *(Complete)*
5. **Design OS v4 assimilation** – Assimilate staged design-system docs, token/mock/registry contracts, package provenance, and evidence. Storybook/Playwright scaffolds, docs-only component stories, and reviewer-inspected candidate visual screenshots are in place; final tokens, product-canon registry adoption, runtime migration, and approved baselines remain gated by separate Point-lock steps. *(Current gated)*
6. **Builder Enablement OS product lock and QA planning** – GOAL-016 docs lock OpenClaw as a Builder Enablement OS centered on SpecGraph; the `openclaw-builder-enablement-os-handoff-v0.1/` package shell has been assimilated into governed repo paths; GOAL-020A/020B/020C QA docs and no-dependency validators are complete. GOAL-017 through GOAL-019 remain blocked until Point reviews the lock. *(Current gated)*
7. **Feature iteration** – Plan and implement next features only after gated Design OS/Product Lock decisions are resolved and Point approves new product scope.
