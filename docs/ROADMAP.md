# Roadmap

This document provides a high‑level roadmap for the OpenClaw Cooperative Cockpit project. Major milestones include:

1. **Repository setup** – Establish agent workflows, approval and concurrency policies, and validation scripts. *(Complete)*
2. **Static MVP assimilation** – Bring the static HTML/CSS/JS prototype into the repo under `apps/static-mvp/` and record the handoff. *(Complete)*
3. **Static MVP product lock and QA closeout** – GOAL-004 through GOAL-008 normalized the object model, UI mapping, Work Packet/Handoff Packet preview, and QA evidence. *(Complete)*
4. **Canon reconciliation** – Align current-state docs, operating authority, object terminology, readiness semantics, and package manifests so agents can trust the repo. *(Complete)*
5. **Design OS v4 assimilation** – Assimilate staged design-system docs, token/mock/registry contracts, package provenance, and evidence. Storybook/Playwright scaffolds, docs-only component stories, reviewer-inspected candidate visual screenshots, final Claude Design adoption review, and status-language canon are in place. Final tokens, product-canon registry adoption/route reconciliation, runtime migration, `review-blocked` visual semantics cleanup, and approved baselines remain gated by separate Point-lock steps. *(Current gated)*
6. **Builder Enablement OS product lock and QA planning** – GOAL-016 docs lock OpenClaw as a Builder Enablement OS centered on SpecGraph, and Point approved that product lock on 2026-06-04; the `openclaw-builder-enablement-os-handoff-v0.1/` package shell has been assimilated into governed repo paths; GOAL-020A/020B/020C QA docs and no-dependency validators are complete. GOAL-017 through GOAL-019 are product-lock unblocked but remain queued until explicit scoped execution. *(Unlocked / queued)*
7. **Static MVP realistic product journey** – GOAL-021 assimilates the UX upgrade handoff package, adds Landing -> Static Demo Entry -> Project Hub -> Project Initialize before the Workbench, and reframes Workbench as an object-aware editor with Object Outline, focus lenses, right object panel, mock local copilot and readiness queue while preserving static/mock-only safety boundaries. POINT-UX-SIMPLIFICATION now applies Point-approved simplification: Workbench leads with Context / SpecGraph / Handoff Gates, onboarding routes hide from the left rail after project open, top Projects access remains available, and Project Initialize transcript behavior remains static mock data only. *(Complete / needs continued design review)*
8. **Feature iteration** – Plan and implement next features only after gated Design OS/Product Lock decisions, Claude Design open gates, and Point-approved new product scope are resolved.
