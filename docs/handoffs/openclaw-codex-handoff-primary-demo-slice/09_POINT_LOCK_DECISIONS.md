# 09 Point-Lock Decisions

## Primary scenario lock

Default: Product spec creation ending in gated handoff preview.

Reason: It is the clearest proof of OpenClaw value.

## Object model lock

Default: keep the current candidate set.

Any merge, rename, or expansion requires Point lock.

## UI metaphor

Default: Operational Control Center.

Do not shift to chatbot, workflow engine, or Dify clone.

Point must approve any claim that says OpenClaw can execute workflows or orchestrate agents.

## New dependencies

Forbidden unless Point approves.

## Runtime semantics

No runtime execution in static MVP.

## Backend/database/auth

Forbidden unless Point approves.

## Real agent orchestration

Forbidden unless Point approves.

## Ninth page / IA expansion

Forbidden unless Point approves.

## Repo path uncertainty
